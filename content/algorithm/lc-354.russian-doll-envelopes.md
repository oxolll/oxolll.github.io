### 說明
給你一個二維整數陣列 `envelopes`，其中 `envelopes[i] = [wi, hi]` 代表第 $i$ 個信封的寬度與高度。
當且僅當一個信封的寬度和高度都**嚴格大於**另一個信封時，這個信封才能將另一個信封裝進去（就像俄羅斯套娃一樣）。
請回傳你最多能套幾個信封。

### 範例
- **Input**: `envelopes = [[5,4],[6,4],[6,7],[2,3]]`
- **Output**: `3`
- **Explanation**: 最多可以套 3 個信封，組合為 `[2,3] => [5,4] => [6,7]`。

### 限制條件
- $1 \le envelopes.length \le 10^5$
- $envelopes[i].length == 2$
- $1 \le w_i, h_i \le 10^5$

## 💡 2026-05-04 初次提交

### 🎯 演算法分析 (Algorithm Analysis)
這題是標準的二維最長遞增子序列 (2D LIS) 問題。因為信封必須「寬度」與「高度」都嚴格大於才能嵌套，如果直接使用 $\mathcal{O}(N^2)$ 的 DP 絕對會 TLE。
我們必須將問題降維到一維，再套用 $\mathcal{O}(N \log N)$ 的 LIS 演算法。

**破局關鍵：巧妙的自訂排序 (Sorting Trick)**
1. **寬度 `w` 由小到大排序**：保證了由左往右掃描時，寬度是逐漸符合條件的。
2. **寬度相同時，高度 `h` 由大到小排序**：這是最精妙的一步！如果寬度相同，這兩個信封是**不能**互相嵌套的。把高度反過來排，當我們在一維高度陣列上做嚴格遞增的 LIS 時，相同寬度的信封就不可能被重複選取（因為後面的高度比較小，無法接在前面較大的高度之後），從而完美避開了無效的組合。

### 🛠️ 解題思路 (Approach)
1. **排序**：使用 lambda 函式對 `envelopes` 進行排序：`w` 升序、`h` 降序。
2. **一維 LIS (Patience Sorting)**：
   - 排序後，寬度的限制已經被隱式滿足且過濾掉無效組合。我們只需對高度 `h` 求解最長嚴格遞增子序列。
   - 宣告一個動態陣列 `dp` (或稱為 `tails`)。
   - 遍歷所有信封的高度 `h`，利用 `lower_bound` 在 `dp` 陣列中尋找第一個大於等於 `h` 的位置。
   - 如果找不到（代表 `h` 比目前所有的結尾都大），將 `h` 加入 `dp` 尾端，序列長度 $+1$。
   - 如果找到，用 `h` 取代該位置的元素。這是一種貪心策略：**在長度相同的情況下，結尾的數字越小，後面能接上其他數字的潛力就越大**。
3. **回傳結果**：`dp` 陣列的大小即為最大嵌套數量。

### 📊 複雜度分析
- **時間複雜度**: $\mathcal{O}(N \log N)$。排序耗時 $\mathcal{O}(N \log N)$，遍歷陣列並執行 Binary Search 耗時 $\mathcal{O}(N \log N)$。
- **空間複雜度**: $\mathcal{O}(N)$。最壞情況下所有信封皆可嵌套，`dp` 陣列會長到與原陣列相同大小。

### 💻 程式碼實作 (C++)
```cpp
class Solution {
public:
    int maxEnvelopes(vector<vector<int>>& envelopes) {
        // 自訂排序：寬度 (w) 升序，高度 (h) 降序
        sort(envelopes.begin(), envelopes.end(), [](const auto& a, const auto& b){
            if(a[0] != b[0]) return a[0] < b[0];
            return a[1] > b[1]; // 寬度相同時，高度降序以避免同寬度被重複套用
        });

        vector<int> dp;
        for(auto p : envelopes) {
            int h = p[1];
            // lower_bound 尋找第一個 >= h 的位置
            auto it = lower_bound(dp.begin(), dp.end(), h);
            
            if(it == dp.end()) {
                // 如果找不到，代表當前高度 h 可以接在最後面，擴展 LIS 長度
                dp.push_back(h);
            } else {
                // 如果找到，則替換掉原本的元素。
                // 貪心策略：讓該長度下的結尾高度盡可能變小，增加後續套入信封的潛力
                *it = h;
            }
        }

        return dp.size();
    }
};

/*
User Comments & LIS DP 意義修正：

對於這種 O(N log N) 的 LIS 解法，陣列命名為 `dp` 是競技程式中非常標準的做法。
若要賦予更直觀的物理意義，可以將其視為 `tails` 或 `min_heights`。

陣列狀態定義：
dp[i] := 構成長度為 i+1 的遞增子序列中，所有可能的結尾元素裡，「最小的那個高度 h」。

替換邏輯：
1. it == dp.end() (找不到 >= h 的元素)：
   代表當前的 h 比目前所有長度的最小結尾都還要大，
   所以直接把 h 接在目前最長子序列的後面，創造出一個更長 (i+2) 的子序列記錄。
   
2. it != dp.end() (找到第一個 >= h 的元素位置 *it)：
   代表我們可以用更小的 h 來達成同樣長度的子序列。
   將 *it 替換為 h，可以讓這個長度組合的「尾巴」變細，後續就更容易把別的信封套上來！
*/
```
